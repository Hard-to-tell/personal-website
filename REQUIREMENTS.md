# 当前需求与设计约定

更新时间：2026-07-08。

## 项目目标

建立一个低成本、容易恢复、面向中国大陆访客的个人网站，用于保存文章和接收访客留言。代码和内容应长期保存在 GitHub；停止续费域名后，仍能使用默认地址或以后重新部署。

## 已确认偏好

- 站名：在流放地
- 站长昵称：Nemo
- 简介：暂时留空
- 视觉：二次元、偏安静、暗蓝色，不要抽卡式堆叠和过度动画
- 动态：允许缓慢壁纸轮换、星尘和很轻的点击反馈
- 内容：简单文章为主
- 互动：公开留言优先，不做完整论坛
- 账号：访客不注册、不登录；站长使用 Twikoo 管理密码
- 图片：访客暂不上传图片
- 预算：固定成本尽量控制在每年 100 元以内，主要支出预期为域名
- 技术选择：优先使用成熟模板和托管服务，不自己维护传统服务器

## 已实现

- Hexo + Reimu 静态博客
- 首页、归档、关于、友链、留言板
- GitHub Pages 自动部署
- Pages CMS 在线写文章
- Twikoo 匿名留言
- Netlify Functions 后端
- MongoDB Atlas 数据库
- 深浅色模式
- 暗蓝色主题、像素初音光标、安静星尘、小行星带
- 顶部壁纸自动轮换
- 固定头像替换路径

## 暂不实现

- 访客注册和邮箱登录
- 访客之间的私信
- 用户发帖、等级、积分、关注关系
- 图片上传和自建图床
- 传统云服务器
- 私密留言系统

如果未来确实出现明确需求，再单独评估这些功能，不提前增加维护成本。

## 仍需站长决定

- 购买哪个域名；
- 网站简介写什么；
- 关于页面最终内容；
- 是否需要友链申请规则；
- 是否开启留言审核、邮件提醒和更严格的反垃圾配置。

## 完成标准

- 手机和电脑均能访问；
- 文章可通过 Pages CMS 或 Markdown 编辑；
- 访客可匿名留言，站长可管理；
- 密钥不进入 Git 仓库；
- 本地执行 `npm.cmd run clean` 和 `npm.cmd run build` 成功；
- GitHub Actions 部署成功；
- 项目文档足够让新手重新运行、修改和恢复网站。
