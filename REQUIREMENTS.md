## 1. 项目目标

建设一个具有明显二次元个人风格的网站，用于发布简单文章，并为访客提供与站长交流的入口。

网站应当轻量、费用低、容易维护。开发工作主要由 Codex 完成；涉及审美、内容和会明显影响使用方式的选择时，再由站长确认。

## 2. 已确认需求

### 2.1 内容

- 以简单文章为主。
- 暂不需要复杂的分类、积分、等级或论坛板块。
- 文章应方便站长后续自行新增和修改。
- 暂不需要访客上传图片。

### 2.2 视觉

- 整体方向为二次元风格。
- 网站需要有一定辨识度，不能只是未经修改的通用博客模板。
- 可以参考成熟模板和同类网站，但最终需要调整配色、排版、素材和细节，形成自己的风格。
- 视觉设计暂不着急确定，后续先收集参考，再制作页面草图或原型供站长选择。

### 2.3 互动

- 访客只需要和站长互动，不需要访客之间组成完整论坛。
- 期望的互动方式包括：
  - 给站长发送私密消息；或
  - 在网站公开留言。
- 如果登录会显著增加成本和维护复杂度，留言可以不要求登录。
- 暂不需要用户发帖、用户私信其他用户、上传图片等社区功能。

### 2.4 登录

- 如果保留访客账号，使用邮箱和密码登录。
- 当前不要求邮箱真实性验证。
- 优先使用现成的认证和数据服务，不自行开发复杂账号系统。
- 是否真的需要访客账号尚未最终确定。由于访客只与站长互动，第一版可以不做访客登录，用留言表单和私密联系表单代替。

### 2.5 访问对象

- 主要面向中国大陆用户。
- 手机和电脑均应正常访问。
- 部署平台需要重点考虑国内访问速度、稳定性和域名备案要求。

### 2.6 分工

- Codex 负责可完成的设计、开发、测试、部署说明和维护文档。
- 站长具备基本代码基础，可以维护文章和进行少量修改。
- 对不确定且会影响最终效果、费用或维护方式的事项，开发前向站长确认。

## 3. 建议的第一版范围

第一版建议采用“个人博客 + 两种联系入口”，不建设完整论坛。

包含：

1. 个性化首页。
2. 文章列表和文章详情页。
3. 关于我页面。
4. 公开留言区（可选，允许匿名留言）。
5. 私密联系表单，内容只发送或展示给站长。
6. 深色模式、移动端适配等基础体验。
7. 简单的防垃圾信息措施。

暂不包含：

- 访客之间的私信。
- 用户发帖和论坛板块。
- 用户上传图片。
- 积分、等级、关注关系和复杂通知。
- 自建邮件服务器或复杂后台。

## 4. 暂定实现思路

### 4.0 已确定的技术栈

- 网站框架：Hexo 8（静态输出）。
- 视觉主题：Reimu 1.12，采用现成的东方 Project 二次元视觉体系。
- 文章管理：Hexo + 本地 Markdown 文件。
- 代码管理：Git。
- 托管候选：腾讯 EdgeOne Makers 免费版。
- 互动候选：Edge Functions + KV 存储，仅在确认需要留言后接入。
- 用户系统：第一版不做访客注册和登录；仅保留站长管理能力。

该组合的核心目标是减少依赖和长期维护成本。即使停止托管，文章和页面也都保存在仓库中，可以重新构建部署。

### 4.1 网站主体

- 文章采用 `source/_posts` 下的 Markdown 文件保存。
- 页面尽量静态生成，以降低托管成本和停运后的恢复难度。
- 主题配置集中保存在 `_config.reimu.yml`，避免升级 Reimu 时覆盖个性化设置。

### 4.2 留言与私密消息

优先评估现成免费服务，避免自行维护服务器和数据库。

- 公开留言：使用现成评论服务，或使用一个非常小的托管数据库。
- 私密消息：使用表单服务、邮件转发服务，或小型后端接口。
- 如果公开留言和私密消息都不需要永久保存，可以进一步简化。

### 4.3 登录取舍

目前更推荐第一版不提供访客登录，原因如下：

- 访客没有发帖、上传图片或个人主页等必须登录的功能。
- 不验证邮箱的账号容易被批量注册，也无法可靠找回密码。
- 去掉访客账号可以减少数据库、邮件、隐私和安全维护工作。

站长自己的内容管理不一定需要网页后台：文章可以直接通过 Markdown 和 Git 管理。以后确实需要访客账号时，可以在保留现有页面的基础上增加现成认证服务。

## 5. 预算目标

- 目标总成本：每年不超过 100 元，能更低则更好。
- 优先使用免费静态托管、免费额度的表单或数据服务。
- 主要固定费用预计为域名续费；实际费用取决于域名后缀、注册商和续费价格。
- 如果选择中国大陆服务器，服务器费用和备案要求可能使总成本超过目标。
- 最终部署平台和域名方案需要在开发前核实当时价格及国内访问情况。

## 6. 停运与恢复要求

网站必须支持停止付费后保留全部核心代码和文章，并能在以后重新上线。

需要保留：

- Git 代码仓库。
- Markdown 文章和网站配置。
- 原始图片、字体等素材及其授权记录。
- 环境变量名称和部署说明，但不把密码直接写进代码仓库。
- 留言或消息数据的定期导出文件（如果使用外部服务）。

停运时可以停止托管或不再续费域名。以后重新购买域名或选择新平台，按部署文档重新配置即可上线。需要注意：域名过期后可能被他人注册，因此“不续费域名”和“保留原域名以后再用”不能同时保证。

## 7. 尚待确认

以下问题在视觉设计或开发对应功能前再确认：

1. 第一版是否完全取消访客注册和登录。
2. 是否同时需要“公开留言”和“私密联系”，还是只保留其中一种。
3. 私密消息希望发送到邮箱，还是登录一个简单后台查看。
4. 公开留言是否需要站长审核后才显示。
5. 二次元视觉的具体方向，例如清新日系、像素、赛博、可爱角色或复古网页。
6. 是否已经拥有可合法使用的头像、背景图、角色图和字体。
7. 是否坚持使用独立域名，以及偏好的域名后缀。
8. 是否接受非中国大陆部署，以降低费用和免去大陆服务器备案流程。
